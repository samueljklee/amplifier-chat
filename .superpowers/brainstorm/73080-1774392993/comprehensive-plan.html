Information Hierarchy

What to show at each visibility tier, applying progressive disclosure.

Always Visible
Model — what am I talking to? CWD — where am I working? Status — thinking / idle / error Session name — primary identifier Time ago — recency for scanning
On Click / Hover
Bundle — what capabilities? Turn count — session depth Duration — how long running Parent / fork info — lineage Last message preview — quick peek Full CWD path — tooltip on short path
On Demand
Session ID — copy button Queue count — only when non-zero

Session Card Data: What Stays, Moves, or Goes

Detailed breakdown of every data element in the current session card and where it lands in the new design.

Data Current Proposed Rationale
Session name Card row 1 Card row 1 (keep) Primary identifier. Never remove.
Status Unicode icon (text) Colored dot Scannable at a glance. Green=running, amber=connecting, red=error, muted=idle/history.
CWD path Card row 2 (always) Removed when grouped 100% redundant with group header. In chronological mode: show leaf name only (e.g., "amplifier-chat").
Turn / msg count Card row 3 (always) Hover / detail panel Inspection metadata, not navigation. Nobody scans by turn count.
Time ago Missing! Card row 2 (new) Second most useful scanning signal after name. "23 min ago" vs "2 days ago" helps find sessions fast.
Model badge Missing! Card row 2 (new) "Which session was I using opus for?" -- critical for distinguishing sessions.
Last message preview Card row 4 Hover tooltip Low signal in 230px. Truncates poorly (code fragments). Name should be the identifier.
New / Updated badge Card row 3 Card row 2 (keep, inline) Actionable state change. High signal. Keep but inline with meta row instead of dedicated space.
Queue badge Card row 3 Card row 2 (keep, when present) Only appears when non-zero. Keep inline.
Overflow menu (...) Always visible Show on hover Reclaim horizontal space for longer names. Right-click as alternative entry point.

Implementation Priority

Ordered by impact and effort. P0 items are quick wins that should ship first.

Priority Change Effort Impact
P0 Rename "New Session CWD" → "Working Directory" and "Session Metadata" → "Session Info" Trivial Immediate clarity. Two string changes.
P0 Add model name to session metadata panel Low Critical missing information. Data already available from session.
P0 Add "time ago" to session cards, replace turn count as primary meta Low Better session scanning. Data already available (lastUpdated).
P1 Remove CWD row from cards when grouped by directory Low Biggest density win. Removes 24px per card. Show 2x more sessions.
P1 Replace status Unicode icons with colored dots Low Better scannability. Consistent visual language.
P1 Add model badge to session cards Low Distinguish sessions by model at a glance.
P1 Add persistent context bar above chat (CWD + model + status) Medium Biggest UX improvement. Session context always visible.
P1 Add recent directories dropdown to CWD picker Medium Major usability win. Pull from session history.
P2 Move last message preview to hover tooltip Low Recover 24px per card. Still accessible on hover.
P2 Unify sidebar footer (smooth draft → active transition) Medium No jarring mode switch. Consistent visual weight.
P2 Add bundle name + duration to session info panel Low More complete session context.
P2 Show overflow menu on hover only (not always) Trivial More horizontal space for session names.
P3 De-emphasize session ID behind copy button Trivial Cleaner metadata panel.
P3 Context bar expanded/collapsed toggle with progressive disclosure Medium Full metadata without sidebar. Power-user feature.
